# App Basics Moved

This project has been moved to its own repository:

**next-store-shift**

This folder originally contained a learning project whose primary goal was to understand modern Next.js patterns, best practices, and the architectural differences between React and Next.js.

As the project evolved, it became **Version 1 (v0.3.51)** of an e-commerce application focused on architecture, authentication, server-first rendering, caching, security, and feature-based design.

The purpose of this version was to learn and apply modern Next.js engineering practices, including:

* App Router
* Server Components
* Client Components
* Server Actions
* Authentication and sessions
* Redis integration
* Feature-based architecture
* Optimistic UI
* Loading, error, and not-found boundaries
* ISR and caching strategies
* Security-focused patterns
* Clean separation of concerns

Version **v0.3.51** represents the completion of the architecture-focused learning phase.

During development, it became clear that the project had grown beyond a learning exercise. The architecture, separation of concerns, and engineering direction made it a strong candidate for a production-ready application and a portfolio project for professional web development opportunities.

For that reason, the project was moved into its own dedicated repository where it can evolve independently.

Planned future improvements include:

* Real database integration
* Product and order models
* Checkout flow
* Admin dashboard
* Automated testing
* CI/CD pipeline
* Better observability and logging
* Docker support
* Role-based authorization
